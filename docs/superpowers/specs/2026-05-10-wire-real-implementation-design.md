# Wire Real Implementation — Design

**Date:** 2026-05-10
**Status:** In progress
**Trigger:** User uploaded a textbook + asked for an audit; got `STUB_AUDIT` fixture instead of a real LLM-driven audit.

---

## Goal

Replace every stub tool handler and every stub API route with the real implementation specified in `docs/ultraplan-00-architecture.md` and `docs/ultraplan-02-system-prompt.md`. After this change, every Enya tool returns content actually generated by the configured Backboard assistant against the user's real inputs.

## Architecture (per locked plan, not invented here)

- **One Backboard assistant** with the master system prompt loaded by Amin.
- **Per-message tool swapping** — different tool arrays per UI context.
- **Per-message model override** — `claude-sonnet-4-20250514` for generation, `gpt-4o` for quiz JSON, `gemini-2.0-flash` for chat.
- **Per-message memory mode** — `Auto` for state writes, `Readonly` for content gen.
- **Documents at assistant scope** — auto-RAG on every thread.
- **Memory at assistant scope** — student profiles, classroom rosters, teacher prefs.
- **No DB, no @anthropic-ai/sdk, no file-backed JSON.**

Every module exports a typed function `runX(args, ctx) → Promise<TypedResult>` that:
1. Builds a system prompt from role + EAL style card + JSON schema instructions.
2. Calls `runStructuredCall` with the right model override (per architecture §4.4).
3. Parses + zod-validates the response.
4. Returns the typed result (or throws — caller decides whether to fall back to seed).

### C. `shared/lib/tool-handlers.ts` — replace every stub

Each handler becomes a 1–3 line wrapper that calls the matching `shared/lib/ai/*` function with `args` and any context (attachments, studentId).

### D. Direct API routes — point to `shared/lib/ai/*`

Every `app/api/teacher/*` and `app/api/student/*` stub route becomes a thin wrapper that:
1. Validates body with zod.
2. Reads seed if `NEXT_PUBLIC_USE_SEED_FALLBACK === "true"` AND seed exists.
3. Otherwise calls the matching `shared/lib/ai/*` function.
4. Returns the result.

This keeps the seed-fallback toggle working as the demo safety net.

### E. Document upload — poll until indexed

`app/api/backboard/upload-document/route.ts`: after `uploadDocument`, call `waitForDocumentIndexed(documentId, { maxMs: 30_000 })` before returning. Frontend already shows a parsing state; just need the route to not return until RAG is actually ready.

### F. Message route — keep keyword-stub fallback only when key missing

`app/api/backboard/message/route.ts`: leave the existing `if (!assistantId || !apiKey)` guard in place — it's a legit dev-without-keys path. Don't remove it, but document why it exists. The real path (with key set) already routes through `runToolLoop` correctly.

### G. External APIs

- **YouTube** — `https://www.googleapis.com/youtube/v3/search` with `YOUTUBE_API_KEY`. Filter `videoEmbeddable=true`, `safeSearch=strict`, `videoDuration=short`, `maxResults=5`. Then optional `videos.list` for duration. Pick top 3.
- **DALL-E 3** — `openai` SDK (already installed). `model: 'dall-e-3'`, `quality: 'hd'`, `style: 'natural'`, `size: '1024x1024'`. Style-locking prefix from architecture §7.1.
- **ElevenLabs** — `@elevenlabs/elevenlabs-js` to mint a signed conversation URL with the per-conversation persona override constructed in `start_voice_conversation`.

## Memory schema

Student profile written as one memory entry:
```
[student:maya] Grade 3 | Emerging EAL (A1) | interests: butterflies, art, drawing, gardens | culture: Newcomer from Aleppo, Syria; Arabic L1 | goals: ...
```

Classroom written as:
```
[classroom:cls_abc123] name: Mrs. Lee's Grade 3 | students: maya, liam | course: photosynthesis-101
```

Search by prefix: `searchMemories("[student:maya]")` returns the profile entry; the model parses it.

This is intentionally text-shaped (not JSON) because Backboard memory is semantic-search-indexed prose, not structured records.

## Output validation

Every structured output is parsed with zod. Two failure modes:
- Schema-pass — return.
- Schema-fail — log a warning, return `{ ...partial, _validationFailed: true }`. Caller may fall back to seed. **Do not crash the chat.**

JSON extraction tolerates:
- Raw JSON
- ```` ```json ... ``` ```` code fences
- ```` ``` ... ``` ```` un-tagged fences
- Leading/trailing prose around a single JSON object

## Out of scope

- Real OAuth, multi-tenancy, district hierarchies — kept stubbed.
- Real curriculum standards database — `search_curriculum_standards` returns LLM-suggested standards based on a tiny BC G3 hand-curated reference list embedded in the system prompt.
- Persistent file storage of uploaded textbooks — Backboard owns the canonical copy.

## Acceptance

After this change:
1. Upload [energyofanastronaut_-_teacher.pdf](_demo-assets/) → message "audit for Bloom's" → audit JSON contains specific phrasing from the actual PDF (not the photosynthesis fixture).
2. Same upload → "create a Grade 3 course" → outline references actual PDF chapter titles.
3. Maya text lesson and Liam text lesson on the same topic differ by ≥40% Levenshtein (the L4 differential test).
4. Creating a new student via chat appears in subsequent `searchMemories("[student:")` results.
5. Story image gen returns a real DALL-E URL or the curated fallback (never a `null`).
6. YouTube search returns real video IDs (not the 3 hardcoded fallbacks).
7. `_stub: true` does not appear in any production response payload.
